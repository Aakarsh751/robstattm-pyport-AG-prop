\section{Wrapper Design Pattern}
%--------------------------------------------------------------------

All \texttt{robstatpy} wrappers follow the same pattern: accept
NumPy/pandas inputs, convert to R objects via \texttt{rpy2}, call the
\texttt{RobStatTM} function, extract named fields with \texttt{.rx2()},
and return a plain Python dictionary.
The setup below is shared across all wrappers:

\begin{lstlisting}[style=python, caption={Shared rpy2 setup for all robstatpy wrappers}]
import os
import numpy as np
import rpy2.robjects as ro
from rpy2.robjects.packages import importr
from rpy2.robjects import default_converter
from rpy2.robjects.conversion import set_conversion

# Persist conversion context across Jupyter async task boundaries
set_conversion(default_converter)

robstattm = importr("RobStatTM")

def rx(robj, name):
    """Extract named element from an R list."""
    return robj.rx2(name)

def r2py(rvec):
    """Convert R vector to numpy array."""
    return np.array(list(rvec))
\end{lstlisting}

\section{Test Task 1 --- \texttt{rpy2} Wrapper for \texttt{locScaleM}}
%--------------------------------------------------------------------

\subsection{Wrapper Code}

\begin{lstlisting}[style=python, caption={rpy2 wrapper for locScaleM (robust location and scale)}]
def loc_scale_m(x, psi="bisquare", eff=0.95):
    """Robust M-estimator of location and scale via RobStatTM::locScaleM().

    Parameters
    ----------
    x   : array-like
        Univariate numeric data vector.
    psi : str
        Psi-function family: 'bisquare' (default) or 'huber'.
    eff : float
        Target asymptotic efficiency (default 0.95).

    Returns
    -------
    dict with keys:
        'mu'     : robust location estimate
        'std_mu' : standard error of mu (sandwich formula)
        'disper' : M-scale of centred residuals
    """
    r_x   = ro.FloatVector(x)
    r_out = robstattm.locScaleM(r_x, psi=psi, eff=eff)
    return {
        "mu":      rx(r_out, "mu")[0],
        "std_mu":  rx(r_out, "std.mu")[0],
        "disper":  rx(r_out, "disper")[0],
    }
\end{lstlisting}

\subsection{Wrapper Code}

\begin{lstlisting}[style=python, caption={rpy2 wrapper for scaleM (robust M-scale)}]
def scale_m(x, family="bisquare", delta=0.5):
    """Robust M-scale via RobStatTM::scaleM().

    Parameters
    ----------
    x      : array-like
        Univariate numeric data vector.
    family : str
        Psi-function family: 'bisquare' (default) or 'huber'.
    delta  : float
        Breakdown-point parameter (default 0.5).

    Returns
    -------
    float : robust M-scale estimate
    """
    r_x = ro.FloatVector(x)
    return float(robstattm.scaleM(r_x, family=family, delta=delta)[0])
\end{lstlisting}

\subsection{Relationship between \texttt{scaleM} and \texttt{locScaleM}}

\texttt{locScaleM} and \texttt{scaleM} are complementary:
\texttt{locScaleM} computes a joint robust location--scale estimate
$(\hat\mu, \hat\sigma)$ using IRLS, and its \texttt{disper} output is
the M-scale of the centred residuals $x_i - \hat\mu$:
\[
  \texttt{locScaleM}(x)\texttt{[`disper`]}
  \;=\; \texttt{scaleM}(x - \hat\mu)
\]
Calling \texttt{scaleM}$(x)$ directly on raw data gives a different
value because it does not first subtract a robust location estimate.
Use \texttt{locScaleM} for a joint estimate; use \texttt{scaleM}
when only a robust spread measure is needed (e.g.\ as a building
block inside another estimator).
